# NeteaseDynamicWatcher

用于监控指定网易云音乐用户动态的本地程序。

## 当前功能

- 监控指定用户动态
- 识别歌曲分享动态
- 首次运行只保存当前状态，不发送历史通知
- 后续新动态通过 PushMe 通知
- SQLite 本地去重
- 本地保存动态媒体资源
- 本地 HTML 档案阅读页面
- 支持 Windows 长时间运行

## 安全说明

- Cookie、通知密钥只保存在本地配置或环境变量。
- 不提交真实账号信息。
- 离线测试使用模拟响应。
- 本地档案页面不依赖网易云在线访问。

## 本地使用流程

1. 复制 `.env.example` 为 `.env`。
2. 填入自己的网易云 Cookie 和 PushMe key。
3. 首次运行：

```text
python run_watcher.py --once
```

首次运行只建立基线。

4. 后续长期运行：

```text
python run_watcher.py
```

默认每 15 分钟检查一次。

## 查看保存的动态

生成本地阅读页面：

```text
python tools/open_ui.py
```

页面会打开本地 HTML，读取 SQLite 中保存的动态以及已经归档的媒体资源。

如果只想查看已有保存内容，不重新请求媒体：

```text
python tools/open_ui.py --skip-archive
```

## 媒体保存规则

- 图片、头像等资源保存到本地。
- 同一个文件内容只保存一份，页面可复用同一资源。
- 动态记录保留原始字段，便于后续增加展示能力。
- 历史头像按动态快照保存，不会被后来的头像覆盖。

## 手动验证

建议先使用 PushMe 自己的测试功能确认手机通知链路正常，再运行监控程序。

## 注意

网易云页面需要登录才能访问动态内容，因此 Cookie 失效后需要更新本地配置。
